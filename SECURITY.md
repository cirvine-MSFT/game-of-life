# Security Policy

## Supported scope

Security reports are accepted for the current `main` branch, GitHub Actions workflows, build and release packaging, and files shipped from this repository.

This project is a prototype. Published release artifacts should be treated as demonstration builds unless a release explicitly says otherwise.

## Reporting a vulnerability

Please do not open a public issue or pull request for a suspected vulnerability.

Use GitHub private vulnerability reporting for this repository:

https://github.com/cirvine-MSFT/game-of-life/security/advisories/new

If private reporting is unavailable, contact the maintainer out of band through the GitHub profile instead of disclosing the issue publicly.

## Disclosure expectations

The maintainer will triage reports privately, decide whether the issue is in scope, and publish an advisory or release note when an actionable fix is available. Please wait for maintainer confirmation before discussing the issue publicly.

## Pull request policy

Unsolicited security fix pull requests are not accepted. Report the issue privately first so the maintainer can coordinate the fix without exposing details in a public branch or CI run.
